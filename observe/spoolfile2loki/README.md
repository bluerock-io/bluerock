# spoolfile2loki

A Go service that reads NDJSON spool files produced by acoustic-lite sensors and forwards events to Grafana Loki.

## Features

- **Single File or Directory Mode**: Monitor a single spool file or a directory of rolling spool files
- **Flexible Timestamps**: Accepts unix milliseconds (integer) or ISO 8601 / RFC 3339 strings
- **Automatic Label Extraction**: Extracts `origin`, `event_type`, `event_name`, and `sensor_id` from event metadata as Loki labels
- **Byte-Offset Tracking**: In directory mode, tracks file offsets to efficiently resume without re-reading
- **Completed File Detection**: Skips fully consumed files on subsequent polls
- **Batch Processing**: Sends events to Loki in configurable batches
- **Mock Mode**: Replays embedded sample data with timestamps shifted to now — no sensor required
- **Graceful Shutdown**: Handles SIGTERM and SIGINT signals

## Building

```bash
just build
```

This builds a multi-arch container image (x86_64 + arm64) tagged as `spoolfile2loki:<version>` and `spoolfile2loki:latest`.

## Running

The container is designed to run as part of the docker-compose stack in `../deploy/`. See the [parent README](../README.md) for deployment instructions.

To run standalone:

```bash
docker run -v /path/to/config.yaml:/etc/spoolfile2loki/config.yaml \
  -v ~/.bluerock/oss-events:/data/spool:ro \
  spoolfile2loki:latest
```

## Configuration

See `config.yaml.example` for a fully annotated reference. Key settings:

```yaml
loki_endpoint: "http://localhost:3100"

# Directory of rolling NDJSON spool files
spool_dir: "/data/spool"

# Or a single spool file (mutually exclusive with spool_dir)
# spool_file: "/path/to/acoustic-lite.json"

# Replay embedded test data (no spool file/dir needed)
# mock: true

poll_interval: 60s
batch_size: 100
state_file: "/var/spoolfile2loki/state"
log_level: "info"

labels:
  environment: "production"
```

## Spool File Format

Each line is a JSON object:

```json
{"ts": 1710000000000, "event": {"meta": {"name": "dns_query", "origin": "sensor-01", "sensor_id": 42911, "type": "network"}, ...}}
```

| Field | Description |
|-------|-------------|
| `ts` | Unix milliseconds (integer) or ISO 8601 string |
| `event` | Arbitrary JSON payload forwarded as the Loki log line |
| `event.meta.name` | Becomes `event_name` label |
| `event.meta.origin` | Becomes `origin` label |
| `event.meta.sensor_id` | Becomes `sensor_id` label |
| `event.meta.type` | Becomes `event_type` label |

## Architecture

```
cmd/spoolfile2loki/main.go   CLI entrypoint (cobra)
internal/
  config.go                  Config loading and validation
  forwarder.go               Core polling loop, file/dir processing, label extraction
  loki.go                    Loki HTTP push client with retry
  logger.go                  Zap structured logger
  state.go                   Persistent byte-offset and completed-file tracking
  mockdata.go                Embedded acoustic-lite sample events
  version.go                 Build version (generated by justfile)
```

## State Persistence

The forwarder maintains a state file to track progress across restarts:

- **Single file mode**: last forwarded timestamp per file
- **Directory mode**: byte offset into the current file, plus a set of completed files (by name and size)

If the state file is deleted, the forwarder reprocesses all available data.
